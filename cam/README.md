# Камеры Санкт-Петербурга

Небольшой веб-проект с интерактивной картой Санкт-Петербурга, списком публичных камер и встроенным live-плеером.

## Что уже есть

- карта на `Leaflet` с маркерами камер;
- поиск по месту, району и описанию;
- боковой каталог камер;
- панель просмотра live-трансляции;
- каталог камер, переведенный на прямые iframe-источники без YouTube;
- удобный запуск через браузер вне Codex.

## Структура

- `index.html` — основная страница;
- `styles.css` — оформление интерфейса;
- `src/cameras.js` — каталог камер;
- `src/app.js` — логика карты, фильтров и плеера;
- `start.command` — быстрый запуск на macOS.

## Как открыть вне Codex

Проект не требует сборки. Есть два простых варианта:

### Вариант 1. Через приложение на macOS

1. Откройте папку проекта.
2. Дважды кликните `Open Cameras.app`.
3. Откроется браузер на адресе `http://127.0.0.1:8080`.
4. Не закрывайте окно терминала, пока сайт нужен.

Приложение уже лежит в корне проекта и просто запускает `start.command`, поэтому отдельная сборка не нужна.

### Вариант 2. Через Finder на macOS

1. Откройте папку проекта.
2. Дважды кликните `start.command`.
3. Откроется браузер на адресе `http://127.0.0.1:8080`.
4. Не закрывайте окно терминала, пока сайт нужен.

### Вариант 3. Через терминал

```bash
python3 -m http.server 8080
```

После этого откройте `http://localhost:8080`.

## Как поделиться ссылкой с друзьями

Самый удобный вариант — GitHub Pages. В проект уже добавлен workflow
`.github/workflows/deploy-pages.yml`, который публикует сайт при каждом пуше в `main`.

### Что сделать один раз

1. Создайте репозиторий на GitHub и загрузите туда проект.
2. В репозитории откройте `Settings` → `Pages`.
3. В поле `Source` выберите `GitHub Actions`.

### Команды для публикации

```bash
git add .
git commit -m "Prepare GitHub Pages deployment"
git branch -M main
git remote add origin https://github.com/<your-username>/<your-repo>.git
git push -u origin main
```

После выполнения workflow сайт будет доступен по ссылке:

`https://<your-username>.github.io/<your-repo>/`

## Почему видео теперь работают лучше

Раньше каталог использовал ссылки вида `youtube.com/embed/live_stream?...`, и часть из них уже не отдавала рабочий эфир.

Сейчас проект переведен на прямые iframe-источники, которые уже используются самими страницами камер:

- `vkvideo.ru/video_ext.php`
- `rutube.ru/play/embed/...`
- `lk-b2b.camera.rt.ru/...`
- `open.ivideon.com/embed/...`

Это не гарантирует вечную доступность каждого потока, но в целом такой подход заметно надежнее для публичного каталога камер.

## Как расширять каталог

Каждая камера описывается объектом в `src/cameras.js`:

```js
{
  id: "unique-id",
  name: "Название камеры",
  district: "Район или зона",
  location: "Более точное место",
  lat: 59.93,
  lng: 30.31,
  sourceName: "Источник",
  sourceUrl: "https://example.com/camera-page",
  embedUrl: "https://provider.example/embed/stream-id",
  tags: ["центр", "трафик"],
  description: "Краткое описание",
  accessNote: "Комментарий по доступности"
}
```

## Важное ограничение

Формулировка "все доступные камеры Санкт-Петербурга" зависит от трех вещей:

- есть ли у источника публичный доступ;
- разрешает ли источник встраивание на внешний сайт;
- можно ли стабильно получить координаты и ссылку на эфир.

Поэтому текущая версия — это рабочий агрегатор публичных камер. Следующий шаг для полноценного покрытия — подключить отдельный реестр источников и периодическую проверку, какие iframe-потоки еще живы.
